# 真實世界業務邏輯反向拆解提示詞 2026-04-26

這份文件是給會切換不同大腦的使用者使用的「萬用提示詞」。用途是讓新的 AI 或工程協作者不要一進來就只看程式碼，而是先反向理解真實生意，再把系統、資料表、頁面、角色、文件和自動化規則對回去。

目前建議先放在 repo 文件中，不做成 skill。原因是這份提示詞仍然高度依賴 OrderSome、宇聯、來點什麼、大永這個專案的語境；放在 `docs/` 並由 `CLAUDE.md` 指向，最容易跟著專案更新。等這套方法穩定到可以跨專案使用，再抽成 Codex skill。

## 使用時機

- 完成大永 Dayone 後，要繼續拆宇聯後台 ERP
- 要理解來點什麼品牌、門市、加盟、電商、配送、帳務的聯動
- 要建立或檢查 ERP、CRM、BOM、採購、庫存、財務、報表系統
- 要讓下一個大腦知道「不要只修頁面，要理解真實營運閉環」
- 要做 UAT、真資料驗證、流程圖、系統頁面對照表

## 萬用提示詞

```text
你現在要接手這個專案，請不要只從程式碼或頁面功能開始。

請先用「真實世界業務運作」反向拆解這家公司或這個模組：

1. 這家公司怎麼賺錢？
   - 誰付錢？
   - 為什麼付錢？
   - 成本從哪裡來？
   - 毛利、現金流、應收、應付分別怎麼發生？

2. 真實世界每天有哪些角色？
   - 老闆 / 管理者
   - 行政
   - 倉管
   - 司機或外場人員
   - 門市或加盟主
   - 客戶
   - 供應商

3. 每個角色每天做哪些動作？
   請用一張流程圖或心智圖表示，並在每個動作旁邊標示：
   - 對應系統頁面
   - 對應資料表或主要資料
   - 對應文件或表單
   - 如果系統已自動處理，標示「系統自動處理」
   - 如果目前沒有對應頁面，標示「無」

4. 請把流程拆成以下幾條線：
   - 銷售線：接單、出貨、送達、收款、AR
   - 進貨線：採購、收貨、驗收、入庫、AP
   - 庫存線：入庫、出庫、退貨、盤點、異動紀錄
   - 帳務線：應收、應付、現金、月結、對帳
   - 客戶線：CRM、客戶等級、聯絡人、交易歷史
   - 商品線：商品、BOM、成本、售價、毛利
   - 權限線：誰可以看、誰可以改、誰需要審核

5. 請檢查系統是否形成閉環：
   - 有訂單是否一定有對應配送或履約狀態？
   - 有送達是否一定有 AR？
   - 有收現是否同步更新訂單與 AR？
   - 有供應商簽收是否一定有 AP？
   - 有入庫是否一定有庫存異動？
   - 有退貨或回庫是否先待驗，再由管理者確認？
   - 有報表數字是否能追回原始單據？

6. 請輸出以下成果：
   - 一句話說明這個業務怎麼賺錢
   - 一張心智圖
   - 一張每日工作流程圖
   - 一張「真實動作 / 系統頁面 / 文件 / 系統是否自動處理」對照表
   - 一張「已驗證 / 未驗證 / 高風險 / 下一步」清單
   - 一段給下一個大腦看的交接摘要

7. 請用非工程師也看得懂的語言。
   如果要引用程式碼，請用「這代表現場會發生什麼」來翻譯。
   不要只說 mutation、router、table，要說它對應的是接單、扣庫、收款、付款、入庫、回庫或對帳。

8. 在做任何修補前，請先確認：
   - 現在連的是哪個資料庫
   - 正式站和本地站的環境變數是否一致
   - 真資料是否可以安全讀寫
   - 是否會影響既有資料
   - 是否需要 rollback 測試

9. 如果要實作，請優先補閉環斷點：
   - 漏 AP
   - 漏 AR
   - 庫存直接回補但未審核
   - 派車重複扣庫或未扣庫
   - 收款只更新訂單、不更新帳務
   - 報表數字不能追單據

請先產出業務地圖，再產出技術任務。
```

## 給大永使用的補充語境

大永相對單純，核心是蛋品批發配送：

- 進貨：跟供應商拿蛋，供應商簽名後產生 AP
- 入庫：貨回到倉庫後才增加可賣庫存
- 銷貨：客戶下單，派車配送
- 收款：送達後產生 AR，司機可現場收款
- 回庫：剩貨先進回庫待驗，管理者確認後才回補庫存

已知主要頁面：

| 業務 | 頁面 |
|---|---|
| 總覽 | `/dayone` |
| 訂單 | `/dayone/orders` |
| 客戶 | `/dayone/customers` |
| 商品 | `/dayone/products` |
| 派車 | `/dayone/dispatch` |
| 進貨收貨 | `/dayone/purchase-receipts` |
| 庫存 | `/dayone/inventory` |
| AR 應收 | `/dayone/ar` |
| 司機 | `/driver/*` |
| 客戶入口 | `/dayone/portal/*` |

## 給宇聯與來點什麼使用的補充方向

宇聯和來點什麼會比大永複雜，不能只套大永的蛋品配送模型。下一階段要拆：

- ERP：採購、庫存、配送、財務、報表
- CRM：客戶、加盟主、門市、聯絡紀錄、商機與合約
- BOM：商品組成、原料、成本、耗損、製程
- 門市營運：日報、排班、維修、稽核、教育訓練
- 加盟體系：加盟主、權利金、返利、貨款、對帳
- 電商：商品、訂單、金流、物流、會員
- 內容與品牌：官網、文章、SEO、活動、表單

拆解時一律先問：

- 這個數字從哪張單據來？
- 這個動作是誰做的？
- 做完後下一個人要看到什麼？
- 系統有沒有自動幫他做完？
- 如果中間失敗，錢、貨、帳會不會不一致？
