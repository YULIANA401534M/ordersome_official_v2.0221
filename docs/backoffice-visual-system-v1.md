# Backoffice Visual System v1

適用範圍：
- Dayone 後台
- 宇聯後台
- 來點什麼後台
- 後續新增的管理型介面、儀表板、營運工作台

目的：
- 建立一致的高級感與可讀性
- 避免每個頁面各自長得不一樣
- 讓後續任務能直接沿用同一套字體、顏色、畫布與元件節奏

## 1. 視覺方向

關鍵字：
- 乾淨
- 高級
- 穩定
- 有溫度
- 適合長時間閱讀與操作

避免：
- 過度裝飾字體
- 高飽和撞色
- 大片沒有資訊密度的空白卡片
- 為了設計感犧牲表格與表單可讀性

## 2. 標準字

角色分工：
- 品牌字 / 展示字：只保留在品牌識別、少量標題、封面區塊
- 後台主標題：以穩定的 UI 字體為主，不使用過度強烈的品牌字
- 內文 / 表格 / 表單：一律使用 UI 字體
- 數字 / 編號：使用 monospace 或穩定數字字型

建議規則：
- 頁面主標題：`font-ui`，800，字距略收緊
- 區塊標題：`font-ui`，700
- 內文：`font-ui`，400-500
- 訂單編號 / 單號 / 系統碼：`font-mono`

原因：
- 後台需要耐看與長時間操作
- 品牌字適合點綴，不適合大面積反覆出現

## 3. 標準顏色應用

核心色盤：
- 主背景：暖白、米白、淺石色
- 主文字：石墨黑、暖灰黑
- 次文字：石灰色
- 主強調：amber / honey / gold 系
- 狀態色：
  - 成功：emerald
  - 警示：amber
  - 危險：red
  - 資訊：sky / blue

使用比例建議：
- 70% 中性色與背景色
- 20% 白色或半透明白卡片
- 10% 強調色

應用規則：
- 主色只用在 CTA、重點數字、目前 active 狀態、重要標籤
- 不同狀態要靠色相區分，但飽和度保持克制
- 大面積區塊不要直接鋪滿高飽和 amber

## 4. 標準畫布

畫布原則：
- 後台不是品牌頁，重點是資訊清楚但不單調
- 需要有空氣感，但不能空到失去內容重心

建議配置：
- 全頁背景：暖白到米色的漸層
- 局部點綴：淡 amber 光暈或柔和 radial gradient
- 內容容器寬度：`1180px - 1220px`
- 區塊間距：24 / 28 / 32 為主

卡片原則：
- 圓角：26-34px
- 邊框：極淡暖灰或 amber tint
- 陰影：柔和、低對比、偏大型
- 背景：白色或暖白半透明，不做厚重玻璃感

## 5. 標準元件節奏

頁首 Hero：
- 一個 eyebrow
- 一個穩定的大標題
- 一段不超過 2-3 行的說明
- 2-4 個摘要卡

篩選列：
- 與資料表分開
- 放在柔和卡片中
- 欄位高度、圓角、間距一致

資料表：
- 保持平穩，不做過多視覺噪音
- 表頭背景淡石色
- row hover 只做很輕的暖色過渡
- 金額、狀態、編號的視覺層次要清楚

手機卡片：
- 保留桌面資訊結構，但改成 2-3 層閱讀節奏
- 第一層：主資訊
- 第二層：次資訊
- 第三層：操作

## 6. 後續實作規則

新增後台頁面時至少要定義：
- 主標題與副標題
- hero 或主資訊入口
- 篩選列樣式
- 空狀態樣式
- 桌面表格與手機卡片對應方式

優先沿用：
- `client/src/index.css` 的 backoffice / dayone token
- `DayoneLayout`
- 共用卡片、表格、toolbar class

## 7. 不同品牌如何共用

共用層：
- 畫布
- 間距
- 卡片質地
- 表格節奏
- 表單規則

品牌層：
- Logo
- 品牌色主 accent
- 少量展示標題
- 模組命名與 icon 氣質

建議方式：
- 先有一套共用 backoffice system
- 再用 tenant / brand token 做少量換皮

## 8. 本輪先定下來的結論

從這一版開始：
- 後台主標題不再依賴過度強烈的品牌字
- amber 是重點色，不是全畫面底色
- 高級感靠比例、留白、陰影、字重與資訊節奏，不靠花樣
- 所有後台後續任務都應優先引用這份規範
