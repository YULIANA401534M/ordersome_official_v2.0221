# Ordersome 財會系統架構文件
> 建立日期：2026-04-08 | 狀態：規劃中

## 核心原則
1. 所有金額變動留 Log（誰改、改什麼、何時）
2. 刪除改為「作廢」，資料永不真正刪除
3. 自動化只做「建議」，人工做「確認」
4. 每個數字都能追溯到源頭單據
5. 重要操作需二次確認，超額需 manager 審核
6. 月底鎖帳，鎖後只能用調整傳票

## 內外帳分工
- 內帳：Ordersome 系統負責（自動化核心）
- 外帳：外包會計師事務所
- 每月提供給會計師：銷售/進貨/費用/薪資明細（CSV匯出）

## 模組清單

### M1 應收帳款 AR
自動化：訂單→應收、逾期計算、對帳單產生
人工確認：收款登錄、爭議處理、壞帳標記、退貨沖銷

### M2 應付帳款 AP
自動化：進貨驗收→應付、成本更新
人工確認：付款登錄、供應商對帳、價格異動記錄

### M3 出納
自動化：司機代收對單、日結計算
人工確認：銀行水單核對、現金盤點、異常差異
多銀行帳戶：每個帳戶餘額分開追蹤

### M4 進銷存成本
自動化：進貨→庫存+成本、出貨→庫存減少、毛利計算
連動：採購→驗收→應付→庫存→成本→訂單→應收→毛利分析

### M5 薪資連動（宇聯集團）
流程：Apollo HR匯出 → Ordersome計算 → 應付薪資 → 出納付款 → 成本分攤
計算項目：本薪+加班+全勤+業績-請假-勞健保

### M6 固定資產
- 設備/車輛/機器折舊自動計算
- 折舊自動進入損益

### M7 預算管理
- 月/季費用預算設定
- 實際 vs 預算差異分析
- 超預算警示

### M8 管理報表（內帳核心）
- 客戶欠款排行 + 逾期警示
- 供應商付款排行
- 月營收/毛利/費用
- 現金流量（實際 vs 預測）
- 未來30天收支預測（現金缺口預警）
- 各品牌/部門損益
- 集團合併損益

### M9 憑證管理
- 每筆帳附上憑證（發票/收據/合約）
- 支援圖片/PDF上傳
- 會計師查帳用

### M10 退貨/爭議處理
- 退貨沖銷原始應收
- 破蛋理賠記錄
- 爭議帳款專屬狀態
- 全程Log留存

### M11 合約與價格歷史
- 客戶合約價格存檔
- 價格變動歷史保留
- 合約到期提醒續約

### M12 壞帳管理
- 長期未付款提列壞帳準備
- 壞帳沖銷流程

### M13 集團內部交易
- 宇聯向各品牌收取權利金/月租費
- 集團合併報表時內部交易沖銷

### M14 稅務時間提醒
- 每月5號：上月營業稅資料準備
- 每年1月底：扣繳憑單
- 每年5月：營利事業所得稅申報

### M15 內控與權限
- 會計：可記帳，不可審核自己的帳
- 出納：只看自己負責現金
- Manager：完整報表
- 記帳與審核必須分離
